# Median of Medians

  • 典型: 中央値は二分探索
  • 典型: 区間和の高速数え上げはBIT
  • 典型: 以上/より小さいが問題になっていているときは二値(-1/+1, 0/1)に変換する

問題

長さNNNの数列の、N(N+1)2\frac{N(N+1)}{2}​2​​N(N+1)​​個の連続部分列に対し、その列それぞれから中央値をとってきて、新しく作った数列の中央値を求める。(この問題における中央値の定義は問題参照)

  • 考察

中央値の性質

数列の中に、値がXXX以上の要素が半数以上存在する。このような性質を満たすXXXのうち最大のものを中央値と呼ぶ。

性質から分かること

数列の中央値をMMMとしたとき、数列の中に値がXXX以上の要素が半数以上存在するならば、X≤MX\leq MX≤M

部分列ひとつを考える

N(N+1)2\frac{N(N+1)}{2}​2​​N(N+1)​​個の部分列のひとつに着目する。値がXXX以上の要素がこの中に半数以上存在する場合をcase1, 半数以上存在しない場合をcase2とする。
この部分列の中央値をMMMとおく。性質から分かることより、case1ではX≤MX \leq MX≤M、case2ではM<XM<XM<Xとなる。

部分列を集める

各部分列の中央値を集め、数列M1,M2,...,MN(N+1)2M_1, M_2, ... , M_\frac{N(N+1)}{2}M​1​​,M​2​​,...,M​​2​​N(N+1)​​​​とする。この数列の中央値MMMを求めたい。
部分列ひとつひとつを考えることにより、それぞれのMiM_iM​i​​がcase1, case2いずれなのか分かっているとする。
すると、case1の個数とcase2の個数をそれぞれC1,C2C_1, C_2C​1​​,C​2​​と置くと、
C1<C2C_1 < C_2C​1​​<C​2​​ならばこの中にcase1すなわちX≤MiX \leq M_iX≤M​i​​なるMiM_iM​i​​が半数以上存在することはないので、 XXXは中央値になることはない。したがって、M<XM<XM<X
C2≤C1C_2 \leq C_1C​2​​≤C​1​​ならばこの中にcase1すなわちX≤MiX \leq M_iX≤M​i​​なるMiM_iM​i​​が半数以上存在するので、 XXXは中央値になりうる。したがって、X≤MX \leq MX≤M
あとはXXXを動かして境界の値であるMMMを求めればよい。

struct BIT {
    long long n;
    vector<long long> dat;

    void initialize(long long n_input){
        n = n_input + 1;
        dat.resize(n+1);
        for(int i = 0; i<=n; i++) dat[i] = 0;
    }

    long long sum(long long i){
        long long s = 0;
        while(i >= 0){
            s += dat[i];
            i = (i & (i+1)) - 1;
        }
        return s;
    }

    void add(long long i, long long x){
        while(i <= n){
            dat[i] += x;
            i |= i+1;
        }
    }
};

  • 解説

  • initialize: n_input+2個の要素を持つvectorを0初期化する

  • sum: a0+a1+...+aia_0+a_1+...+a_ia​0​​+a​1​​+...+a​i​​を求める。

  • add:

  • i&(i+1)について
